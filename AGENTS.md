# Portfolio Repository Guidance

## Shipping changes

- After completing and validating an in-scope code change, commit it and push it to the current remote branch without waiting for a separate request.
- Stage only files that belong to the completed task, and preserve unrelated user changes.
- Do not push partial work, known failing changes, secrets, or credentials.
- If pushing is blocked by authentication, branch protection, conflicts, or remote failures, report the blocker and leave the validated commit intact locally.
